# 06 · Backtracking

## What this pattern covers

Exploring every possible combination/permutation by building a solution step by step, and "undoing" (backtracking) as soon as a branch stops being viable, to avoid exploring more than necessary.

## What to recognize in the problem statement

- Asks to **generate all possible combinations, permutations, or subsets** satisfying a condition.
- Words like "all ways to", "all solutions", "generate each..." appear, instead of "find one/the best".
- There are **constraints that allow pruning** entire branches before fully exploring them (sudoku, N-queens, valid partitions).
- Differs from dynamic programming (folder 07) in that here you usually need **all** valid solutions, not just the optimal value of one.
