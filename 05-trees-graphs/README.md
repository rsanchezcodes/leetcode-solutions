# 05 · Trees & Graphs

## What this pattern covers

DFS traversals (recursive or with an explicit stack) and BFS (with a queue) over trees and graphs, to explore structures, find paths, connected components, or the shortest path in unweighted graphs.

## What to recognize in the problem statement

- The input structure is explicitly a **tree** (binary or n-ary) or a **graph** (adjacency list/matrix, or a grid that can be treated as an implicit graph).
- Asks for the **depth, height, or a path** within a tree — usually recursive DFS.
- Asks for the **shortest path** or minimum number of steps in an unweighted graph — usually BFS, not DFS.
- Asks for **connected components**, "islands" in a matrix, or whether two nodes are connected.
- Explicitly mentions "level order" traversal — BFS with a queue.
