# 07 · Dynamic Programming

## What this pattern covers

Solving a problem by breaking it down into overlapping subproblems with optimal substructure, storing already-computed results instead of recomputing them — via memoization (top-down) or tabulation (bottom-up).

## What to recognize in the problem statement

- Asks for the **minimum/maximum value or number of ways** to do something, not all explicit solutions (that would be backtracking, folder 06).
- A "naive" recursive solution would **recompute the same subproblem many times** (e.g. Fibonacci, subsequences).
- You can clearly define a **state** (which parameters uniquely identify a subproblem) and a **transition** (how a state's solution is built from smaller states).
- Typical keywords: "minimum number of steps/coins/jumps", "length of the longest subsequence", "number of ways to reach...".
